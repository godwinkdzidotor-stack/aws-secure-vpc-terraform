Secure AWS VPC Reference Architecture

A Terraform-based AWS VPC design featuring public and private subnets, routing through an internet gateway and NAT gateway, and security controls that support segmented, production-style deployments.

Terraform AWS VPC Security
Network Layout
Topology
  • Single VPC with well-defined CIDR range.
  • Multiple public subnets across availability zones.
  • Multiple private subnets for application and data tiers.
High Availability Zonal Resilience
Routing Components
Connectivity
  • Internet gateway attached for public subnet egress.
  • NAT gateway for private subnets to reach the internet securely.
  • Route tables defined and associated per subnet type.
IGW NAT Gateway
Security Groups
Instance Level
  • Separate groups for bastion, web, and application tiers.
  • Inbound rules limited to least privilege.
  • Outbound rules restricted based on instance role.
Least Privilege
Network ACLs
Subnet Level
  • Baseline NACLs aligned with subnet purpose.
  • Deny rules for disallowed ranges where needed.
  • Complements security groups without duplication.
Deploying with Terraform
Usage
  1. Clone the repository and configure AWS credentials.
  2. Adjust variables.tf for CIDRs, region, and naming.
  3. Run terraform init to pull providers.
  4. Run terraform plan to preview changes.
  5. Run terraform apply to deploy the VPC.
